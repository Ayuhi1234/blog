I wrote about why this exists in more depth [here](/blog/rewrote-blog-three-times). The short version: the third rewrite of this site was built on one rule — content lives in files, not a database — and that rule has made publishing sustainable in a way the previous two versions never were.

## Stack notes

Next.js App Router for static generation and routing, `content-collections` for a typed MDX pipeline, and Tailwind for a design system built around a blue-purple-emerald accent palette instead of the grayscale-plus-one-accent look most templates default to.

## What I'd change

The search index is currently built at request time from all content — fine at this content volume, would need a proper index (or a hosted search service) well before it became a problem at scale.
